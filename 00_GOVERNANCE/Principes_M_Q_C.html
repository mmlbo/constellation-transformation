📊 Principes M/Q/C

🔴 Classe M – Mandatory (indispensable)
Tâches sans lesquelles la transformation ne peut réussir. Exécution obligatoire pour passage de phase. Ex: migration Siècle Food, déploiement ERP.
🟠 Classe Q – Quality-dependent (ajustable)
Tâches nécessaires mais paramétrables. Le COPIL peut ajuster les seuils de qualité. Ex: SLA CSP (80% cible → 70% acceptable).
🟢 Classe C – Continuous improvement (non bloquant)
Tâches d'amélioration continue. Peuvent être décalées sans impact critique. Ex: BI avancé, rolling forecast.

Règle d'arbitrage

Si une tâche M est bloquée (RAG Rouge), le système recommande de geler les tâches Q et C dépendantes pour libérer des ressources.
